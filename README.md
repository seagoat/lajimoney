# 可转债折溢价套利系统

可转债折溢价套利机会扫描与方案分析系统，支持持仓对冲、融券对冲和裸套三种策略模式。

---

## 核心原理

### 什么是可转债

可转债（Convertible Bond，CB）是上市公司发行的债券，持有人可以在约定时间内按约定价格将债券转换为公司股票。

### 折溢价套利基本逻辑

可转债具有"债性"和"股性"双重属性：
- **债性**：作为债券持有，到期还本付息
- **股性**：可以按转股价转换为正股

**转股价值** = 正股价格 ÷ 转股价 × 100

> 例如：正股价格 10 元，转股价 8 元，则每张转债的转股价值 = 10 ÷ 8 × 100 = 125 元。

**溢价率** = （转债价格 - 转股价值）÷ 转股价值 × 100%

| 情况 | 含义 |
|------|------|
| 溢价率为正 | 转债价格 > 转股价值，转债偏贵 |
| 溢价率为负（折价） | 转债价格 < 转股价值，存在套利空间 |

**折价空间** = 转股价值 - 转债价格

当折价存在时，买入转债 → 立即转股 → 卖出正股，可以获得价差收益。

### 三种套利策略

#### 策略一：持仓对冲（HEDGE）

用户持有正股底仓时的套利方式。优先使用此策略，风险最低。

1. **扫描机会**：检查持仓正股对应的转债是否出现折价（溢价率 < 阈值）
2. **买入转债**：买入张数根据底仓股数自动换算
3. **立即转股**：将持有的转债按转股价转换为正股
4. **卖出正股**：卖出对冲现有持仓，锁定价差收益

**目标张数计算**：
```
目标张数 = ceil(持有股数 × 转股价 ÷ 100 ÷ 10) × 10
```
例如：持有 1000 股，转股价 10 元 → 需买入 100 张 CB（10 手）

#### 策略二：融券对冲（SHORT）

用户无正股底仓、但券商支持融券时的套利方式。隔夜风险小。

1. **扫描机会**：全市场寻找折价转债（需在交易所融资融券标的名单中）
2. **融券卖出正股**（做空），获得现金
3. **买入转债 + 立即转股**
4. **T+1日**：用转股获得的正股归还融券
5. **剩余现金 - 融券利息 = 净收益**

**目标张数计算**（按资金上限）：
```
目标张数 = floor(融券资金上限 ÷ (CB价格 × 100))，向下取整到10的倍数，至少10张
```

#### 策略三：裸套（NAKED）

用户无正股底仓、也无法融券时的套利方式。存在隔夜风险。

1. **扫描机会**：全市场寻找折价转债（需要更大折价作为安全垫）
2. **买入转债 + 立即转股**
3. **次日开盘卖出正股**

**目标张数计算**（按资金上限）：
```
目标张数 = floor(裸套资金上限 ÷ (CB价格 × 100))，向下取整到10的倍数，至少10张
```

**风险提示**：正股次日开盘价波动可能导致亏损。2% 折价作为安全垫需要次日正股跌幅 < 2% 才能盈利。

#### 策略优先级

**底仓对冲 > 融券对冲 > 裸套**。系统根据用户底仓和券商标的配置自动选择最优策略。

## 关键公式

```
转股价值  = 正股价格 ÷ 转股价 × 100
溢价率    = (转债价格 ÷ 转股价值 - 1) × 100%
折价空间  = 转股价值 - 转债价格

转股获得股数 = 目标张数 × 100 ÷ 转股价
```

### 预估成本与收益计算

#### 持仓对冲（HEDGE）

```
CB买入成本     = CB价格 × 目标张数
转股卖出所得   = 正股价格 × 转股获得股数
CB双向佣金     = CB买入成本 × 佣金率 × 2
股票卖出佣金   = 转股卖出所得 × 佣金率
印花税         = 转股卖出所得 × 印花税率
预估总成本     = CB买入成本 + CB双向佣金 + 股票卖出佣金 + 印花税
预估收益       = 转股卖出所得 - 预估总成本
预估收益率     = 预估收益 ÷ CB买入成本 × 100%
```

#### 融券对冲（SHORT）

```
CB买入成本     = CB价格 × 目标张数
融券卖出所得   = 正股价格 × 转股获得股数
CB买入佣金     = CB买入成本 × 佣金率
融券卖出佣金   = 融券卖出所得 × 佣金率
预估融券利息   = 融券卖出所得 × 日利率 × 1天
预估总成本     = CB买入成本 + CB买入佣金 + 融券卖出佣金 + 预估融券利息
预估收益       = 融券卖出所得 - 预估总成本
预估收益率     = 预估收益 ÷ CB买入成本 × 100%
```

#### 裸套（NAKED）

```
CB买入成本     = CB价格 × 目标张数
股票卖出所得   = 正股价格 × 转股获得股数
CB买入佣金     = CB买入成本 × 佣金率
股票卖出佣金   = 股票卖出所得 × 佣金率
预估总成本     = CB买入成本 + CB买入佣金 + 股票卖出佣金
预估收益       = 股票卖出所得 - 预估总成本
预估收益率     = 预估收益 ÷ CB买入成本 × 100%
```

## 系统功能

### 首页扫描

点击"手动触发扫描"，系统通过 SSE 流式展示扫描进度：

- **持仓模式**：仅扫描底仓正股对应的转债，速度快（几秒）
- **全市场模式**：扫描全市场所有有效转债（耗时约 20-60 秒）

**扫描结果**以统一表格展示全部被扫描的转债（不仅是套利机会），包含：
- 转债/正股代码名称
- 底仓标注（有/无）
- 策略类型（对冲/融券/裸套/无机会）
- 溢价率、折价空间
- 目标买入价、目标张数
- 预估成本、预估收益、预估收益率
- 风险等级、操作步骤

**三维筛选**：按策略类型 / 溢价率 / 底仓筛选

**表头排序**：点击任意列标题可升序/降序排序

### 底仓管理

在"底仓管理"页面添加持有的正股（系统只做记录，实际持仓由用户在券商账户管理）。

### 信号记录

查看历史扫描产生的套利机会信号（状态：PENDING / EXECUTED）。

### 扫描设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 折价率阈值 | -1.0% | 溢价率低于此值触发套利信号 |
| 每次买入张数 | 10 | 无底仓时的默认张数 |
| 扫描模式 | 仅持仓正股 | 或全市场转债 |
| 股票券商佣金率 | 万1（0.0001） | 买卖均收 |
| 股票印花税率 | 千1.5（0.0015） | 仅卖出时收取 |
| 转债券商佣金率 | 万0.6（0.00006） | 买卖均收 |
| 裸套折价阈值 | -2.0% | 裸套专用，需更大折价安全垫 |
| 启用裸套模式 | true | 底仓为空时是否启用裸套 |
| 启用融券对冲模式 | false | 无底仓时优先于裸套 |
| 融券模式单次最大资金 | 10万元 | 按资金上限计算目标张数 |
| 裸套模式单次最大资金 | 10万元 | 按资金上限计算目标张数 |
| 保证金比例 | 50% | 融券卖出市值 × 保证金比例 = 所需保证金 |
| 日利率 | 0.022% | 融券利息，按日计息 |
| 强制平仓线 | 130% | 保证金低于此比例时券商强平 |

## 系统要求

- Python 3.10+
- Windows / macOS / Linux
- 网络连接（实时价格数据请求）

## 快速启动

### Windows

```batch
.\run.bat
```

### macOS / Linux

```bash
./run.sh
```

## 技术架构

- **后端**：FastAPI + SQLite (aiosqlite)
- **前端**：Jinja2 模板 + 原生 JavaScript
- **数据源**：新浪财经（实时价格）+ 东方财富（转债基础信息）+ akshare（融资融券标的）
- **环境管理**：uv（首次运行自动安装）

## 项目结构

```
app/
  main.py              # FastAPI 入口
  database.py          # SQLite 初始化与迁移
  models.py            # Pydantic 数据模型
  config.py           # 数据库路径配置
  routers/
    holdings.py       # 底仓管理 API
    signals.py         # 信号扫描与 SSE 流式 API
    trades.py         # 成交记录 API
    summary.py         # 汇总统计 API
    settings.py        # 设置 API
    margin.py          # 融券持仓 API
  services/
    data_fetcher.py    # 新浪 + 东方财富 + akshare 数据获取
    signal_engine.py   # 信号计算引擎（三种策略）
    trade_engine.py   # 模拟成交执行
    margin_engine.py   # 融券利息计算
  templates/
    base.html          # 页面基模板
    index.html         # 首页（扫描监控 + 机会报告）
    holdings.html     # 底仓管理
    signals.html      # 信号记录
    trades.html       # 成交记录
    settings.html      # 扫描设置
run.py                 # uvicorn 启动脚本
run.bat                # Windows 启动脚本
run.sh                 # macOS/Linux 启动脚本
requirements.txt       # Python 依赖
```
