# 扫描机会报告 - 设计文档

## 背景

当前系统的"扫描"功能在发现套利机会后直接执行模拟买入并写入 `trades` 表。
用户希望改为：**只扫描机会、生成交易方案和风险评估报告，不执行实际交易**。
后续有券商交易接口时再对接执行层。

## 核心改动

### 1. 设置页面 (`/settings`)

新增两个设置项：

| 字段名 | 说明 | 默认值 |
|--------|------|--------|
| `short_max_fund` | 融券模式单次最大资金（元） | 100000 |
| `naked_max_fund` | 裸套模式单次最大资金（元） | 100000 |

对应的 `settings` 表 + `SettingsUpdate` Pydantic 模型 + 前端表单同步更新。

现有设置项保持不变：折扣阈值、每次买入张数、扫描模式（仅持仓/全市场）、券商费率等。

### 2. 扫描逻辑 (`signal_engine.py`)

三种策略的**目标买入张数**计算方式：

| 策略 | 触发条件 | 目标张数计算公式 |
|------|----------|----------------|
| **HEDGE**（对冲） | 底仓中有该正股 | `ceil(持有股数 × 转股价 ÷ 1000) × 10` |
| **SHORT**（融券） | `is_marginable=True` 且 `short_enabled=True` | `floor(融券资金上限 ÷ 正股价格 ÷ 100) × 100 ÷ (100 ÷ 转股价)`，最终换算为张数（向下取整） |
| **NAKED**（裸套） | `naked_enabled=True` | `floor(裸套资金上限 ÷ CB价格 ÷ 100) × 100 ÷ (100 ÷ 转股价)`，最终换算为张数（向下取整） |

> 注：`short_available` 字段仅表示是否在交易所融资融券标的名单中（理论可融），不代表实际可融数量。实际可融数以券商账户数据为准。

**扫描流程不变**，但扫描结果不再写入 `trades` 表，仅记录到 `signals` 表（状态为 `PENDING`，不再自动改为 `EXECUTED`）。

### 3. 报告展示（首页 `/`）

扫描完成后，在首页直接展示**统一格式的机会报告表格**，不再有"模拟买入"动作。

#### 报告表格列

| 列名 | 说明 | 适用 |
|------|------|------|
| 转债 | 代码 + 名称 | 全部 |
| 正股 | 代码 + 名称 | 全部 |
| 策略 | HEDGE / SHORT / NAKED | 全部 |
| 溢价率 | premium_rate | 全部 |
| 折价空间 | discount_space | 全部 |
| 目标买入价 | 建议 CB 买入价 | 全部 |
| 目标张数 | 计算出的张数 | 全部 |
| 预估成本 | CB 买入金额 + 各项费用估算 | 全部 |
| 预估收益 | 估算净利润 | 全部 |
| 预估收益率 | 收益 / 成本 % | 全部 |
| 风险等级 | 高（HEDGE无）/ 中（NAKED）/ 低（SHORT对冲） | 全部 |
| 操作步骤 | 简要文字描述 | 全部 |

> 注：预估成本和预估收益的计算逻辑见下方"费用与收益估算"章节。

#### 前端展示逻辑

- 策略类型用不同颜色标签：HEDGE=绿色，SHORT=蓝色，NAKED=橙色
- 风险等级用颜色：高=红色，中=橙色，低=绿色
- 扫描结果仍保留在 `signals` 表，供后续"查看信号"页面查阅
- 不再自动生成 `trades` 记录

### 4. 费用与收益估算

#### HEDGE（对冲）
- **预估成本** = CB买入金额 + CB双向佣金 + 股票卖出佣金 + 印花税
  ```
  CB买入金额 = buy_price × 100 × target_shares
  CB双边佣金 = CB买入金额 × cb_broker_fee × 2
  股票卖出金额 = CB买入金额（近似）
  股票卖出佣金 = 股票卖出金额 × stock_broker_fee
  印花税 = 股票卖出金额 × stock_stamp_tax
  ```
- **预估收益** = 股票卖出金额 - CB买入金额 - 全部佣金 - 印花税
- **预估收益率** = 预估收益 / CB买入金额 × 100%

#### SHORT（融券）
- **预估成本** = CB买入金额 + CB买入佣金 + 融券卖出佣金 + 预估融券利息
  ```
  CB买入金额 = buy_price × 100 × target_shares
  CB买入佣金 = CB买入金额 × cb_broker_fee
  转股获得股数 = 100 / conversion_price × target_shares
  融券卖出金额 = stock_price × 转股获得股数
  融券卖出佣金 = 融券卖出金额 × stock_broker_fee
  预估融券利息 = 融券卖出金额 × margin_daily_interest × 预估持仓天数(1天)
  ```
- **预估收益** = 融券卖出金额 - CB买入金额 - CB买入佣金 - 融券卖出佣金 - 预估融券利息
- **预估收益率** = 预估收益 / CB买入金额 × 100%

#### NAKED（裸套）
- **预估成本** = CB买入金额 + CB买入佣金 + 股票卖出佣金
  ```
  CB买入金额 = buy_price × 100 × target_shares
  CB买入佣金 = CB买入金额 × cb_broker_fee
  转股获得股数 = 100 / conversion_price × target_shares
  股票卖出金额 = stock_price × 转股获得股数
  股票卖出佣金 = 股票卖出金额 × stock_broker_fee
  ```
- **预估收益** = 股票卖出金额 - CB买入金额 - CB买入佣金 - 股票卖出佣金
- **预估收益率** = 预估收益 / CB买入金额 × 100%

### 5. 操作步骤文字描述

每种策略生成简要操作指南：

- **HEDGE**：`买入CB → 转股 → 卖出正股（对冲现有持仓）`
- **SHORT**：`买入CB → 申请融券卖出正股 → 转股归还融券`
- **NAKED**：`买入CB → 转股 → 次日卖出正股（注意隔夜风险）`

### 6. 风险等级说明

| 策略 | 风险等级 | 说明 |
|------|----------|------|
| HEDGE | 低 | 有底仓对冲，无隔夜风险 |
| SHORT | 低 | 融券对冲，隔夜风险小 |
| NAKED | 高 | 裸套存在隔夜风险（正股+CB双重波动） |

## 数据流

```
用户点击"扫描"
  → signal_engine.scan_all_cb_gen() / scan_portfolio_gen()
     → 三种策略判断 → 计算目标张数
     → 返回 signal_found step（含全部字段）
  → 前端 SSE 收集所有 signal_found
  → 扫描完成时计算预估成本/收益/收益率/风险等级/操作步骤
  → 首页渲染统一报告表格
  → signals 表写入记录（status=PENDING）
  → trades 表不写入（不执行模拟买入）
```
